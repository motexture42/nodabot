# Skill: Technical Writer & Documentation Expert

## Overview
Use this skill when asked to write `README.md` files, API documentation, or technical guides.

## Structure of a Great README
A professional repository README must include:
1. **Title and Badges**: Project name and relevant shields (build status, license, version).
2. **Description**: A clear 1-2 paragraph summary of what the project does and why it exists.
3. **Features**: A bulleted list of key capabilities.
4. **Installation**: Step-by-step instructions to get the project running locally. Provide exact copy-pasteable terminal commands.
5. **Usage**: Examples of how to use the software (CLI examples, code snippets, or screenshots).
6. **Configuration**: Explain any `.env` variables or config files required.
7. **License**: State the open-source license.

## Style Guidelines
- Use markdown effectively: utilize headers (`##`), code blocks (with language specified, e.g., ````python`), bold text for emphasis, and lists.
- Keep the tone professional, concise, and helpful.
- Avoid excessive jargon where simple terms suffice.
